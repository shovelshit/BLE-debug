# BLE Debug - iOS 使用文档

一款 iOS 蓝牙调试工具，支持扫描设备、读写特征值、快捷操作，并可结合 iOS 快捷指令实现自动化控制。

---

## 安装说明

### 工具准备

| 工具 | 用途 | 下载 |
|------|------|------|
| **iLoader (Mac)** | 安装 IPA | [[GitHub Releases]](https://github.com/nab138/iloader/releases) |
| **iLoader (Windows)** | 安装 IPA | [[GitHub Releases]](https://github.com/nab138/iloader/releases) |
| **SideStore** | 续签（可选） | [sidestore.io](https://sidestore.io) |

> ⚠️ **当前已知问题**：SideStore 登录 Apple ID 存在 Bug，暂时无法通过 SideStore 直接安装。  
> 请按以下步骤使用 **iLoader 直接安装 IPA**。

---

### 安装步骤

#### 1. 安装 iLoader

根据你的系统下载对应版本：

- **Mac**：下载 `iLoader-mac.dmg`，安装后打开
- **Windows**：下载 `iLoader-win.exe`，安装后打开

用 **USB 线**将 iPhone 连接到电脑，iLoader 会自动识别设备。

#### 2. 安装 IPA

在 iLoader 中选择 **「安装 IPA」**，选取以下文件：

```
ios/build/ipa/BLEDebug.ipa
```

等待安装完成，iPhone 桌面出现「BLE Debug」图标即为成功。

#### 3. 信任开发者证书

首次安装需要手动信任证书：

**设置 → 通用 → VPN 与设备管理 → 开发者 APP → 信任**

---

### 签名有效期说明

使用免费 Apple ID 签名，证书有效期为 **7 天**，到期后 App 无法打开。

到期后有两种续签方式：

- **iLoader 重新安装**：重新用 iLoader 安装一次 IPA 即可，操作与首次安装相同
- **SideStore 续签**：打开 SideStore → 找到 BLE Debug → 刷新签名

## 使用说明

### 功能概览

```
扫描页  →  连接车辆蓝牙设备
设备页  →  查看服务和特征值
特征值页 →  读写数据、生成快捷操作
快捷操作 →  一键执行 BLE 写入
日志页  →  查看收发数据记录
```

---

### 第一步：扫描并连接车辆设备

1. 打开 App，进入 **「扫描」** 页
2. 点击右上角扫描按钮，等待设备列表出现
3. 找到车辆的蓝牙设备（通常以车辆编号命名）
4. 点击设备名称 → **连接**

连接成功后自动跳转到设备页，可以看到所有蓝牙服务（Service）列表。

---

### 第二步：找到目标特征值

1. 在设备页点击对应的 **Service** 展开
2. 找到支持 **Write（写入）** 的特征值（标签为 `W` 或 `WNR`）
3. 点击特征值进入 **特征值详情页**
4. 可以在此页面手动输入数据测试读写

> 💡 车辆解码通常使用特定的 HEX 指令，建议先用已知指令测试特征值是否响应。

---

### 第三步：生成快捷操作

在特征值详情页，点击右上角 **「添加到快捷操作」**：

| 字段 | 说明 |
|------|------|
| **操作名称** | 自定义名称，如「解锁」「开坐桶」 |
| **设备 UUID** | 点击「填入当前」自动填写 |
| **写入数据** | 选择 HEX 格式，填入指令数据 |
| **等待响应** | 开启后会等待设备 Notify 回包并展示结果 |

保存后在 **「快捷操作」** 页可见，点击「执行」按钮可直接测试。

---

### 第四步：配置 iOS 快捷指令（关键步骤）

结合 **钱包中的任意卡片**（交通卡、银行卡等），用手机 NFC 触碰车辆 NFC 感应位置自动触发解码。

> ⚠️ **前提**：iPhone 需已在「钱包」App 中添加至少一张卡片，并支持 NFC 功能（iPhone 7 及以上）。

#### 创建自动化

1. 打开 iPhone **「快捷指令」** App，进入 **「自动化」** 标签
2. 点击右上角 **「+」** → **「个人自动化」**
3. 选择 **「钱包」** → 选中钱包中的任意卡片 → 点击「下一步」
4. 依次添加以下动作：

| 顺序 | 动作 | 说明 |
|------|------|------|
| 1 | **等待 5 秒** | ⚠️ 必须添加，等待车辆蓝牙开机完成 |
| 2 | **执行 BLE 快捷操作** | 选择之前创建的快捷操作（如「解锁」） |
| 3 | **通知**（可选） | 显示执行结果，如「解码成功」 |

5. 关闭「运行前询问」，点击「完成」

#### 使用流程

```
手机 NFC 卡片触碰车辆 NFC 感应位置
        ↓
自动触发快捷指令
        ↓
等待 5 秒（等待车辆蓝牙开机）
        ↓
BLE Debug 在后台连接蓝牙并发送指令
        ↓
（可选）推送通知提示执行结果
```

> 💡 **提示**：等待时间可根据车型调整，部分车型蓝牙开机更快，可缩短至 3 秒。

---

### 快捷操作执行结果说明

执行完成后会展示结果弹窗：

| 字段 | 说明 |
|------|------|
| 发送数据 (HEX) | 实际发送的字节数据 |
| 响应数据 (HEX) | 设备回包数据（需开启「等待响应」） |
| 响应数据 (ASCII) | 回包的可读文本形式 |

在 iOS 快捷指令中，这些字段均可作为变量传递给后续动作。

## 常见问题

**Q：连接后找不到特征值？**  
A：点击设备页右上角刷新，重新发现服务。若仍找不到，尝试断开重连。

**Q：关闭蓝牙再打开后，执行快捷操作报错？**  
A：正常，App 会自动重新连接，稍等片刻或手动在扫描页重新连接设备。

**Q：快捷指令执行超时？**  
A：确认车辆蓝牙已开启，设备在有效范围内（约 10 米内）。App 扫描超时默认为 10 秒。

**Q：证书 7 天到期后 App 打不开？**  
A：用 iLoader 重新安装 IPA，或用 SideStore 续签，两者均可恢复使用。
